Sales & Returns Module — Annotated Screen Mockup

Reference wireframe — not a design to copy. This shows the required data elements, actions, and behaviors. You choose the layout, components, and styling. Your screen will look different from this — that’s expected.

Sales Navigation

These buttons appear across the top of every sales screen. The user can move freely between screens without losing data — the system keeps everything in memory.

  [ Customer Search ]  [ Product Selector ]  [ View Cart ]  [ Checkout ]  [ Cancel ]
    ↑ Always visible across all 4 sales screens
    ↑ Cancel resets everything (customer, cart, coupon) — warns before clearing

The four sales screens share a single in-memory sale. Navigating between screens does not discard data. Cancel is destructive — it clears the customer, the cart, and any coupon. Always warn first.

Sale Completion

  User clicks Complete Sale
       │
       ├── Customer selected?         ──→ NO: button stays disabled
       ├── At least one item in cart?  ──→ NO: button stays disabled
       ├── Payment type chosen?        ──→ NO: button stays disabled
       │
       └── ALL YES ──→ Complete the sale
                       │
                       ├── QOH decreases by qty sold (per item)
                       ├── Sale ID generated and displayed
                       ├── All buttons disabled (sale locked)
                       └── Screen shows: "Sale Completed — Sale ID: 12345"

After completion, the user clicks OK and returns to Customer Search with a clean slate.

Sales Button State Matrix

Element Building Sale Sale Completed

Cancel Enabled — warns if data entered Disabled
Complete Sale Disabled until prerequisites met Disabled
Coupon Verify / Clear Enabled Disabled
Cart quantity fields Editable Disabled
Add / Delete in cart Enabled Disabled

Complete Sale Prerequisites (all three required)

  • Customer selected
  • At least one item in the cart
  • Payment type chosen (Cash, Debit, or Credit)

D2 Security: Returns are restricted to Sales Manager role only.

Sales Calculation Formulas

Extended Price = Quantity × Selling Price    (per cart row)

Subtotal = SUM(Extended Price)               (all cart rows)
Discount = Subtotal × CouponPercent          (only if coupon verified)
GST      = (Subtotal - Discount) × 0.05
Total    = (Subtotal - Discount) + GST

Discount line only appears in totals when a coupon has been verified. Otherwise the display shows Subtotal, Tax, and Total only.

Disabled Return Fields

A return quantity field is disabled (grayed out, no input allowed) when either condition is true:

Condition Example Above Why
Part is non-refundable (Refundable = N) Brake Oil, pint Store policy — these items cannot be returned
All purchased units already returned (Orig Qty − Returned = 0) O-ring gasket (2 − 2 = 0) Nothing left to return

When disabled, both the Qty to Return field AND the Reason field are locked.

Element No Invoice Loaded Invoice Loaded (editing) Return Processed
Search Enabled Enabled — warns if data entered Enabled (new search)
Clear Enabled Enabled — warns if data entered Enabled

Cancel Enabled Enabled — warns if data entered Disabled
Qty to Return fields N/A Editable (where not disabled) Disabled

Process Return Prerequisites (all required)

  • A valid invoice is loaded
  • At least one refundable item has Qty to Return > 0
  • Every item with Qty to Return > 0 has a Reason entered
  • No Qty to Return exceeds (Original Qty − Previously Returned)

After processing: Confirmation shown, QOH updated, all fields locked, Return ID displayed.

Return Calculation Formulas

Return Subtotal = SUM(Qty to Return × Original Price)    (per returned item)

Discount        = Return Subtotal × Original Discount %   (from the original sale —
                                                           honored even if coupon expired)

Tax             = (Return Subtotal - Discount) × 0.05

Total Refund    = (Return Subtotal - Discount) + Tax

Worked Example (from wireframe above)

Returning: 1 Helmet ($90.00) + 1 Fuel Filter ($23.95)
Original sale discount: 10%

Return Subtotal = (1 × $90.00) + (1 × $23.95) = $113.95
Discount        = $113.95 × 0.10              = $11.40
Tax             = ($113.95 - $11.40) × 0.05   = $102.55 × 0.05 = $5.13
Total Refund    = $102.55 + $5.13             = $107.68

Refund method: Cash (matches original payment method)

Previously Returned — How It Works

The “Returned” column is not just from one prior return. It is the total across every return ever processed against this invoice.

  Invoice #2000 — Helmet (originally purchased: 5)

  Return #1 (last month):  returned 2 helmets
  Return #2 (last week):   returned 1 helmet
                           ─────────────────
  "Returned" column shows: 3

  Max returnable now: 5 − 3 = 2

What the System Must Never Do

  • Never let a sale proceed without a selected customer
  • Never let a sale complete without at least one cart item and a payment type
  • Never allow cart quantity to exceed QOH
  • Never allow QOH to go negative (sales or returns)
  • Never apply more than one coupon per sale
  • Never apply an unverified coupon
  • Never allow duplicate rows in the cart (duplicates increase existing qty)
  • Never allow returning non-refundable items
  • Never allow return quantity to exceed (Original Qty − Previously Returned)
  • Never switch customers or search new invoices without warning about data loss
  • Never allow edits after a sale or return is finalized
  • Never process a return without a reason for every returned item
