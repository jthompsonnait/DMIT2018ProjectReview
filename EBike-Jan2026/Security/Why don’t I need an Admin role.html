How “Admin” Works in This System

In this system, Admin is not a separate role you assign in the database. Instead, Admin is treated as a super-role that automatically has access to everything.

That means:

  • You will never see “Admin” listed in [AspNetUserRoles].

  • You do not need to add Admin as a row in the table.

  • Admin users are granted all permissions in code, not by database entries.

Why This Is Done

Rather than assigning Admin to every role manually, the system does this:

If the user is an Admin → they automatically have access to every area and every feature.

So Admin does not depend on role records like:

  • Salesperson

  • Shop Manager

  • Mechanic

  • Store Staff

Admins bypass those checks entirely.

What This Means for You

When you are checking [AspNetUserRoles]:

  • ✅ Only worry about non-admin roles

  • ✅ Assign users to roles based on their job

  • ❌ Do NOT try to insert Admin as a role

  • ❌ Do NOT duplicate Admin into every permission

If someone is an Admin, the application logic already treats them as:

“Has all roles”

No extra database entries required.
