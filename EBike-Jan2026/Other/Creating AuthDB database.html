🔐 Setting Up the Authentication Database with Entity Framework Core

Follow these steps to build your authentication database using Entity Framework Core and your configured ApplicationDbContext.

✅ Step 1: Open the Developer PowerShell

  1. In Visual Studio, go to:
    View → Terminal
  2. In the dropdown (top-right of the terminal panel), choose:
    "Developer PowerShell"

✅ Step 2: Check if Entity Framework Tools Are Installed

In the terminal, run:

dotnet ef --version

If the tools are installed, you’ll see something like:

Entity Framework Core .NET Command-line Tools 9.0.3

Copy

✅ Step 3: Install Entity Framework CLI Tools (If Not Already Installed)

If dotnet ef is not recognized, install it using:

dotnet tool install --global dotnet-ef

You should see:

Tool 'dotnet-ef' (version '9.0.3') was successfully installed.

📝 This step is only required once per machine.

✅ Step 4: Navigate to the Project Folder

From the root of your solution, change into the ProjectWebApp directory:

cd .\\ProjectWebApp\\

Note: You can start typing ProjectWebApp and hit tab to auto-complete it.

This folder contains your Blazor or ASP.NET Core project that includes the ApplicationDbContext.

✅ Step 5: Create the Authentication Database

Run the following command to apply migrations and create the database:

dotnet ef database update --context ApplicationDbContext

This uses the ApplicationDbContext class to generate all the necessary Identity tables (like AspNetUsers, AspNetRoles, etc.) to your new Authentication Database, which is "eBike_Auth" in this project. The database the Auth tables are created in is based on the database name you specify in the connection string for authentication.

Note: The initial connection string was in appsettings.json and named AuthDB

"AuthDB": "Server=xxxx;Database=eBike-Auth;Trusted_Connection=True;TrustServerCertificate=True;MultipleActiveResultSets=true"

You can verify this in your database management tool by seeing the newly created tables.

🎉 You’re Done!

Your authentication database is now created and ready for use. You can start registering users, assigning roles, and implementing authorization in your application.

Example Code for Getting Name/EmployeeID

using Microsoft.AspNetCore.Components;
using Microsoft.AspNetCore.Components.Authorization;

namespace ProjectWebApp.Components.Pages
{
    public partial class Home
    {
        private string fullName = string.Empty;
        private string userID = string.Empty;

        [Inject]
        private AuthenticationStateProvider AuthenticationStateProvider { get; set; } = default!;

        protected override async Task OnInitializedAsync()
        {
            await base.OnInitializedAsync();
            var authState = await AuthenticationStateProvider.GetAuthenticationStateAsync();
            var user = authState.User;

            if (user.Identity is { IsAuthenticated: true })
            {
                fullName = user.FindFirst("FullName")?.Value ?? "Unknown User";
                userID = user.FindFirst("EmployeeID")?.Value ?? "";
            }
        }
    }
}
