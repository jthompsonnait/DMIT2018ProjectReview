### Project Overview

**eBike Application** — Team-based project with 4 subsystems:
- **Purchasing** — Purchase order management (`EBike-Jan2026/Purchase Order/`)
- **Receiving** — Goods receipt processing (`EBike-Jan2026/Receiving/`)
- **Sales & Returns** — Sales transactions and returns (`EBike-Jan2026/SalesReturn/`)
- **Servicing** — Service record management (`EBike-Jan2026/Servicing/`)

**Tech Stack:** C# / .NET / Blazor
**Assessments:** Deliverable 1 (rubric) + Deliverable 2 (rubric)
**Teams:** Each team works on one subsystem

### Essential Rules
- **Source of truth:** Requirements HTML + Functional Mockups per subsystem
- **Marking:** Use rubrics in `EBike-Jan2026/Marking/` as authoritative grading criteria
- **ERDs:** Each subsystem has its own ERD image — reference for database validation
